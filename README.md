# 『マヨイビト』DCG v0.4

自作トレーディングカードゲーム『マヨイビト』を、ブラウザで遊べるようにしたものです。
このv0.3は**プレイテスト用の配布版**で、説明なしで起動から対戦までたどり着けることを目指しています。

## 遊ぶ

インストールは要りません。ブラウザで開くだけです。

**スマートフォンを縦向きで**遊ぶことを想定しています。横向きにすると注意が出ます。

## 遊べるもの

| モード | 内容 |
|---|---|
| チュートリアル | v0.5で追加予定 |
| CPU対戦 | CPUを相手にひとりで遊びます。難易度は5段階 |
| ひとり回し | 1台の端末で両方のデッキを交互に操作します |
| CPU観戦 | CPU同士の対戦を自動で流します（開発者向け） |

デッキは**ヨマモリ村**と**黒薔薇の館**の2つ。同じデッキ同士のミラー対戦もできます。

### CPUの難易度

| 難易度 | 目安 |
|---|---|
| 弱 | カードゲームに不慣れな人向け |
| 中 | 基本的な行動を行う標準的なCPU |
| 強 | 盤面や手札を考えて効率的に動く |
| エキスパート | 高度な判断を行う、本気の対戦用 |
| 理不尽 | CPUがこっそりイカサマします。公平な勝負ではないおまけ |

はじめてなら**弱**から、ルールを覚えたら**強**を試してください。

### 遊び方が分からないとき

モード選択の「遊び方」に、ルールと操作をまとめてあります。
対戦中でも、右側の歯車ボタン → メニュー → 遊び方 から開けます。

はじめての方は、**CPU対戦の「弱」**から始めてください。
慣れてきたら難易度を上げていくと、少しずつ理解が進む作りになっています。
（順を追って学べるチュートリアルは v0.5 で追加予定です）

## 不具合を見つけたら

### 対戦が終わっている場合

リザルト画面の**「対戦結果をコピー」**を押して、出てきた文章をそのまま送ってください。
デッキ・難易度・先攻・**シード**が入っているので、こちらで同じ対戦を再現できます。

### エラー画面が出た場合

**「エラー情報をコピー」**を押して、そのまま送ってください。
何をした直後に起きたかも一緒に書いてもらえると助かります。

### そのほか

「ここが分かりにくい」「この操作がやりづらい」も同じくらい役に立ちます。
勝てなかった対戦のシードを送ってもらえれば、こちらで同じ手札から再現できます。

## 更新したのに変わらないとき

ブラウザが古いファイルを覚えていることがあります。
URLの末尾に `?v=2` のような文字を足して開き直すと、確実に読み直せます。

## 開発者向け

### 構成

```
index.html          画面の骨組み
css/layout.css      見た目
images/             カード画像
js/
  version.js        版の番号（ここ1か所を直す）
  cards.js          カードのデータ
  decks.js          デッキの中身
  random.js         シード付き乱数
  effects.js        カード効果
  game.js           ルール処理（勝敗・ターン・盤面）
  ai-core.js        合法手の列挙と襲撃の予測
  ai-heuristic.js   AIの判断
  ai-deckstack.js   エキスパート・理不尽のイカサマ
  ai-player.js      難易度プロファイル
  ai-uiops.js       効果中の選択にAIが答える
  cpu-driver.js     CPUのターンを1手ずつ見せる
  screens.js        画面遷移と各設定
  preview.js        対戦画面の描画と操作
  howto.js          遊び方の本文
  result.js         リザルトと結果コピー
  save-manager.js   セーブデータの管理
  collection.js     所持カード
  events.js         ゲーム内のできごとの通知
  storage.js        保存の橋渡し（save-manager へ）
  se.js             効果音（合成音）
  assets.js         画像の先読み
  errors.js         不具合の記録
tests/              画面なしで動かす検証
```

### 設計上の約束

- **ルール処理は `game.js` だけが持つ。** 画面もAIもここを通します
- **CPUも人間と同じ関数を呼ぶ。** `Game.playUnit` などを共通で使うので、CPUだけ違う挙動になりません
- **`Math.random()` を対戦結果に使わない。** すべてシードから作るので、同じシードなら同じ対戦になります
- **`village` / `mansion` は座席の名前。** どのデッキを使うかは別に決まります（ミラー対戦のため）

### 検証を走らせる

Node.js があれば、画面なしで確かめられます。

```
node tests/ai-tests.js            AIの判断が意図どおりか
node tests/seat-deck-tests.js     座席とデッキの分離・ミラー対戦
node tests/cpu-driver-tests.js    CPUの進め方
node tests/cpu-hidden-tests.js    CPUの手札が漏れないか
node tests/watch-tests.js         観戦の一時停止
node tests/menu-tests.js          メニュー階層
node tests/save-tests.js          保存・所持カード
node tests/settings-tests.js      設定・効果音・保存
node tests/result-tests.js        リザルトと結果コピー
node tests/robust-tests.js        エラー処理と読み込み
node tests/ui-fix-tests.js        操作まわり
```

### 版を上げるとき

`js/version.js` の `APP_VERSION` と、`index.html` の読み込みに付いている `?v=` を
同じ番号に揃えてください。片方だけ直すと、古いファイルを掴んだままになります。

### 開発用の調整パネル

`js/screens.js` の先頭にある `DEV_PANEL` を `true` にすると出ます。配布版では `false` です。
